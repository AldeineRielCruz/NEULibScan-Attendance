# **App Name**: EduRegister Scan

## Core Features:

- Student Check-in Form: A 'login-like' form for students to record their attendance, capturing instructional email (validated for '@neu.edu.ph' domain), sex (Male/Female), and college program from a dropdown list. This data is recorded along with a timestamp.
- Attendance Record Storage: Secure storage of all student attendance data including email, sex, college program, and exact check-in date and time, to support the admin dashboard.
- Admin Login Portal: A separate tab on the main page for administrators to securely log in using a predefined email ('admin@neu.edu.ph') and password ('adminPassword').
- Admin Dashboard Display: An administrative interface to visualize all collected student attendance data in a clear and organized manner.
- Categorized Attendance Reports: Within the admin dashboard, attendance data can be filtered and displayed by various categories: time (Hour, Day, Month, Year, linked hierarchically), College, and Sex, to provide insightful overviews.

## Style Guidelines:

- The visual design will be based on a scholarly and calming greenish palette, evoking a library or university environment. The primary color will be a deep, authoritative forest green (#2F5F2F), suggesting knowledge and stability. The background will be a soft, light mint green (#ECF6EC), providing a spacious and airy feel without distracting. A vibrant but harmonious yellow-green accent (#93DB74) will be used for interactive elements and highlights to draw attention without being overtly bold.
- Headlines will utilize 'Alegreya', a humanist serif, to convey an elegant, intellectual, and traditional library feel. Body text, forms, and administrative information will use 'PT Sans', a humanist sans-serif, chosen for its modern readability and warmth, ensuring clear communication in both small and large blocks of text.
- Icons will adopt a clean, minimalist line-art style. Imagery will subtly reference a library or university context, such as stylized open books, graduation caps, or minimalist building silhouettes, ensuring a consistent visual metaphor.
- The layout will emphasize clear separation between the student check-in and admin login sections. The student form will be streamlined and focused, while the admin dashboard will feature an organized, modular structure with distinct areas for data display and filtering controls. Both will prioritize readability and ease of navigation across various device sizes.
- Subtle and functional animations will enhance user interaction without creating distractions. These will include gentle fades or slides for transitions between sections, responsive hover effects on interactive elements, and clear visual feedback for form submissions and data loading states.
